MY WORKS

Technical Summative 1

In this project, I compared two e-commerce website, namely, Landers and SnR. I first identified three similarities between the two. Then, I noted three key differences. To illustrate these comparisons, I took screenshots of each website, highlighting the relevant features. I found that Landers’ website offers a better overall design for my personal preference. Its layout is more effective for shopping and product discovery.

Technical Summative 2

In this project, I created an HTML file that uses form and table elements, centered around a Pokémon theme, plus my own little profile. The form allows users to input information about various Pokémon, such as their names, types, birthday, etc. The table is designed to display a list of Pokémon, organized into columns for easy viewing.

Technical Summative 3

In this project, I created a mock FEU announcement page using HTML and CSS. The page states, "Due to unexpected weather changes and for the safety of our students, face-to-face classes will be canceled on May 16 and 17, 2024." It also advises students to check their Canvas accounts for asynchronous assignments and further announcements from their professors.

Technical Summative 4

In this project, I designed a collection of six Pokémon cards, arranged in pairs following the provided guidelines. Each card features elements identification, including an image of the Pokémon, its name, and its type with a Pokemon themed background. I added two buttons below each card: "Add Pokémon" and "Release Pokémon."

Technical Summative 5

In this project, I used CSS Grid and Flexbox to design an interactive tool themed around Teen Titans Go! This tool allows users to adjust Flexbox properties, such as alignment, justification, and spacing, and immediately see the effects of these adjustments in real-time when minimizing or maximizing the window.

Technical Summative 6

In this project, I created a responsive home page about BT21, my favorite characters, using HTML and CSS. The page features images of the characters, texts, and a small store section, avoiding placeholders. I included at least two breakpoints to ensure the design adapts effectively to different screen sizes.

Technical Summative 7

In this project, I created a Flexbox simulator using HTML, CSS, and JavaScript. The simulator features a layout with three cards in a container displayed on the right side, while the control buttons are on the left side. These controls include options to ‘Reset Flexbox,’ adjust ‘Flex Direction,’ modify ‘Align Items,’ and Grow the flexbox.’ This lets users to easily play around with different Flexbox settings and see how the card layout changes.

Midterm Examination

In this project, I created a Pokémon portfolio using CSS Flexbox concepts learned during the first half of the semester. to create a responsive layout. It starts with a four-by-three grid of Pokémon icons, switches to a three-by-four grid on smaller screens, and then transitions to a list format with Pokémon names and a Pokéball icon as the window screen narrows further.
